# AES67 Virtual Soundcard — 使用指南

> 把整台 Windows 变成一块常驻的 AES67 网络声卡：系统声音 → 引擎混音路由 → RTP 组播 → 专业音频网络。

## 1. 项目概览

### 软件栈

```
Qt 面板 ──→ 命名管道 ──→ AES67 引擎 ──→ RTP/UDP 组播 ──→ 第三方 AES67 设备
                │                                 ↑
                └── WASAPI 环回 ←── 虚拟声卡驱动     └── 网络收流 → 虚拟麦克风
```

| 组件 | 目录 | 作用 |
|------|------|------|
| **虚拟声卡驱动** | `driver/` | Windows 里出现一块扬声器，收系统音频 |
| **AES67 引擎** | `engine/` | 常驻服务：混音、路由、RTP 收发、PTP 时钟、SAP 通告 |
| **ASIO DLL** | `asio/` | DAW 软件（Reaper 等）直接接入 AES67 网络 |
| **Qt 面板** | `panel/` | 状态监控、路由矩阵、增益/静音、网络配置 |

### 关键路径

```
发送：应用播放 → AES67Driver 扬声器 → WASAPI 环回 → 引擎混音 → RTP 组播
接收：网络 RTP → 引擎收包 → jitter buffer → 物理扬声器渲染（当前）
                                                            → 虚拟麦克风（计划 P8）
```

> **当前阶段**（P6）：内核驱动是"哑端点"——只注册播放设备、接受音频但不发网络包。发网由用户态引擎通过 WASAPI 环回完成。IOCTL + 共享内存直通通路计划在 P9 打通。

## 2. 开发环境搭建

### 2.1 必需软件

| 软件 | 版本 | 下载 |
|------|------|------|
| **Visual Studio** | 2022 (v17) 或 2026 (v18) | [visualstudio.microsoft.com](https://visualstudio.microsoft.com/) |
| **Windows SDK** | 随 VS 安装 | VS 安装器勾选「Windows SDK」 |
| **Windows WDK** | 与 SDK **同版本** | [aka.ms/wdk/windows11](https://aka.ms/wdk/windows11) |
| **Qt 6** | 6.8.x | [qt.io/download](https://www.qt.io/download) |
| **CMake** | 3.16+ | VS 安装器自带，或 [cmake.org](https://cmake.org/) |
| **Git** | 任意 | [git-scm.com](https://git-scm.com/) |

> ⚠️ **WDK 版本必须和 SDK 版本完全一致**。检查：`C:\Program Files (x86)\Windows Kits\10\Include\` 下目录名（如 `10.0.28000.0`）与 WDK 安装版本一致。

### 2.2 VS 工作负载

安装 VS 时勾选：
- ✅ **使用 C++ 的桌面开发**
- WDK 会自动关联 VS（安装后 VS 项目能识别 `.vcxproj` 的驱动配置）

### 2.3 Qt 安装

1. 下载 Qt 在线安装器
2. 勾选 **Qt 6.8.x → MSVC 2022 64-bit**（或 MSVC 2026 64-bit）
3. 默认路径 `D:\Qt\6.8.3\msvc2022_64`（用其他路径需改 `build.py`）

### 2.4 验证环境

打开 **Developer Command Prompt for VS**，依次验证：

```cmd
where cl         # → 应显示 VS 的 cl.exe 路径
where cmake      # → CMake 路径
dir "C:\Program Files (x86)\Windows Kits\10\Include\*\km"   # → 应有 portcls.h
dir "D:\Qt\6.8.3\msvc2022_64\bin\qmake.exe"                 # → Qt 路径
```

## 3. 编译所有组件

### 3.1 驱动（AES67Driver.sys）

用 VS 打开 `driver\AES67Driver.vcxproj`，配置选 **Win10 / x64**，Ctrl+Shift+B。

或命令行（需先改 `driver\build.bat` 中的 WDK 路径）：

```cmd
cd driver
build.bat
```

产物：`driver\x64\Win10\AES67Driver.sys`、`.inf`

### 3.2 引擎（aes67_engine.exe）

```cmd
cd engine
cl /EHsc /std:c++17 /O2 /nologo /W3 /utf-8 ^
  main.cpp aes67_engine.cpp wasapi_device.cpp audio_thread.cpp ^
  logger.cpp network_thread.cpp sap_announcer.cpp ^
  network_receiver.cpp audio_render_thread.cpp ^
  ptp_clock.cpp ptp_thread.cpp pipe_server.cpp ^
  /Fe:aes67_engine.exe ^
  ole32.lib avrt.lib ws2_32.lib winmm.lib
```

### 3.3 面板（aes67_panel.exe）

```cmd
cd panel\build
cmake -S .. -B . ^
  -DCMAKE_PREFIX_PATH="D:\Qt\6.8.3\msvc2022_64" ^
  -G "Visual Studio 18 2026" -A x64
cmake --build . --config Release
```

产物：`panel\build\Release\aes67_panel.exe`

> Qt 路径和 VS Generator 名按本机实际修改。项目已设 `WIN32`，面板不会弹出控制台黑窗。

### 3.4 一键构建脚本（build.py）

```cmd
python build.py build_driver     # 编译驱动
python build.py build_engine     # 编译引擎 + loopback + sine_test
python build.py build_panel      # 编译 Qt 面板
python build.py build_asio       # 编译 ASIO DLL
```

首次使用前编辑 `build.py` 顶部 `QT_DIR` 和 `VSDIR` 变量。

## 4. 驱动安装

### 4.1 开启测试模式

```cmd
bcdedit /set testsigning on
```

**重启电脑**。

> ⚠️ 如果提示"安全引导策略保护"，进 BIOS **关闭 Secure Boot** 再试。

### 4.2 安装测试证书

```cmd
cd driver\x64\Win10
certutil -addstore Root AES67Driver.cer
certutil -addstore TrustedPublisher AES67Driver.cer
```

### 4.3 签名 Catalog

```cmd
cd driver\x64\Win10

REM 生成 catalog 文件（SDK 版本号按本机改）
"C:\Program Files (x86)\Windows Kits\10\bin\10.0.28000.0\x86\inf2cat.exe" /driver:. /os:10_19H1_X64

REM 签名
"C:\Program Files (x86)\Windows Kits\10\bin\10.0.28000.0\x64\signtool.exe" sign /fd SHA256 /a AES67Driver.cat
```

### 4.4 安装

```cmd
pnputil /add-driver driver\x64\Win10\AES67Driver.inf /install
```

### 4.5 验证

```cmd
pnputil /enum-devices /class MEDIA | findstr AES67
```

应看到 `AES67Driver (WDM)` 设备。打开 Windows **声音设置 → 播放**，能看到 **AES67Driver** 播放端点。

> ⚠️ 刚装完驱动如果声音设置里看不到，运行：
> ```cmd
> net stop audiosrv && net start audiosrv
> ```
> 这会让 Windows 音频服务重新枚举设备。

### 4.6 卸载

```cmd
pnputil /delete-driver oem<编号>.inf
```

编号在安装时的输出里找（`发布名称: oem175.inf`），或用 `pnputil /enum-drivers` 搜 AES67Driver。

### 4.7 驱动安装常见坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 编译报 `portcls.h` 找不到 | WDK 没装/版本不匹配 | 装对应版本 WDK |
| 签名报"不包含数字签名" | 只签了 .sys，INF 没签 | 用 `inf2cat` + `signtool` 签 `.cat` |
| `signtool /a /f xxx.cer` 失败 | .cer 只有公钥无私钥 | 用 `/a` 自动从证书存储选 |
| 装完声音设置看不到设备 | 音频服务没重新枚举 | `net stop audiosrv && net start audiosrv` |
| `devcon` 不管用 | 找不到设备或报错不明确 | 换 `pnputil` |

## 5. 启动引擎

### 面板托管模式（推荐）

引擎初始化后**不自动开始流**，等待面板发 START 命令：

```cmd
E:\jmdev\AES67\engine\aes67_engine.exe --duration 0 --managed --rx
```

| 参数 | 含义 |
|------|------|
| `--duration 0` | 常驻不退出（0 = 手动 Ctrl+C 停止） |
| `--managed` | 面板托管模式：初始化 + 等待面板指令 |
| `--rx` | 启用接收路径（全双工，可听到网络流） |

### 命令行模式（自启动，无需面板）

```cmd
aes67_engine.exe --rx --duration 30
```

30 秒后自动停止。适合快速测试。

### 完整参数列表

```
Usage: aes67_engine.exe [options]

  --duration N      运行时长（秒），0=常驻（默认 10）
  --managed         面板托管模式：初始化后等面板 START
  --no-autostart    同 --managed
  --log FILE        日志输出到文件
  --help            显示帮助

TX（发送）:
  --dest ADDR       RTP 目标地址（默认 239.69.1.128）
  --port N          RTP 目标端口（默认 5004）
  --no-tx           禁用发送

RX（接收）:
  --rx              启用接收
  --source ADDR     源组播地址（默认 239.69.1.128）
  --rxport N        接收端口（默认 5004）

音频格式（锁定 AES67）:
  L24 / 48kHz / 2ch / ptime=1ms
```

## 6. 启动面板

```cmd
E:\jmdev\AES67\panel\build\Release\aes67_panel.exe
```

### 面板功能

| 标签页 | 功能 |
|--------|------|
| **Status** | 引擎状态（运行/停止）、实时指标（帧率、TX/RX 包数、Jitter Buffer、PTP） |
| **Settings** | 网络配置（TX 目标地址/端口、RX 源地址/端口），Apply 即时生效 |
| **Routing** | 目的地（AES67 流）+ 路由矩阵（源通道 → 目标流、增益、静音），Apply Routing 运行时生效 |

### 典型操作流程

1. 启动引擎（`--managed`）
2. 启动面板
3. Status 页看到 "● Stopped" → 点 **Start**
4. 看到 "● Running"，指标开始动 → 音频流通
5. Routing 页调增益/静音 → 点 **Apply Routing** → 立刻生效
6. 关闭面板 → 引擎自动停止（3 秒心跳超时）
7. 面板最小化到系统托盘，右键可退出

## 7. ASIO 驱动（可选）

ASIO DLL 让 DAW（如 Reaper）直接接入 AES67 网络，跳过 WASAPI：

```cmd
cd asio
cl /LD /EHsc /std:c++17 /O2 /nologo /W3 /utf-8 /MD ^
  asio_minimal.cpp /Fe:AES67_ASIO.dll ^
  ws2_32.lib winmm.lib ^
  /link /DEF:asio_exports.def
```

### 注册 ASIO 驱动

1. 管理员运行 `regedit`
2. 导入 `asio\asio_registry.reg`（路径按实际修改）
3. 或手动创建：

```
HKLM\SOFTWARE\ASIO\AES67 Virtual ASIO
  CLSID = {A1B2C3D4-E5F6-7890-ABCD-EF1234567890}
  Description = "AES67 Virtual Soundcard ASIO Driver"

HKLM\SOFTWARE\Classes\CLSID\{A1B2C3D4-...}
  InprocServer32 = <full path>\AES67_ASIO.dll
  ThreadingModel = Apartment
```

> ⚠️ ASIO 时钟与 WASAPI 时钟有 ~50ppm 微偏差，长时间运行 jitter buffer 会耗尽，需 ASRC 或 PTP 同步（计划中）。

## 8. 常见问题

### 引擎

| 问题 | 解决 |
|------|------|
| 引擎启动报"TX disabled: AES67Driver render endpoint not found" | 驱动没装或音频服务没重新枚举（`net stop audiosrv && net start audiosrv`），或者在 Windows 声音设置里把 AES67Driver 设为默认设备 |
| 引擎启动报"RX disabled" | 没有非 AES67Driver 的物理输出设备。接上耳机/扬声器 |
| 面板连不上引擎（"Disconnected"） | 引擎必须用 `--managed` 启动。确认引擎控制台输出 "Pipe server listening" |
| `--rx` 全双工听不到声音 | 检查 Settings 页 RX Source Address 是否匹配发送地址。检查 Windows 音量、物理设备 |
| 静音后再取消没有声音 | **已修复**（`feature/dev-fix` 分支）。确保引擎是最新编译的版本 |
| 运行中改路由不生效 | 点完 **Apply Routing** 按钮，按钮变绿 "✓ Applied" 才表示成功 |

### 驱动

| 问题 | 解决 |
|------|------|
| 开机蓝屏 | 先进安全模式，`bcdedit /set testsigning off`，重启。确认 WDK 版本和 SDK 匹配后重新编译安装 |
| 装驱动报"第三方 INF 不包含数字签名" | Catalog 没签或签名无效。重新 `inf2cat` + `signtool` |
| `inf2cat` 报"DriverVer set to incorrect date" | INF 文件里的 DriverVer 日期必须在当前时间之前。改 `AES67Driver.inf` 中 `DriverVer=` 行的日期 |

### 编译

| 问题 | 解决 |
|------|------|
| `cl` 找不到 | 在 Developer Command Prompt 中运行，或先执行 `vcvars64.bat` |
| `cmake` 找不到 Qt6 | 设置 `CMAKE_PREFIX_PATH` 到 Qt 安装目录 |
| CMake 找不到 VS Generator | Generator 名和 VS 版本对应：VS2022=`"Visual Studio 17 2022"`，VS2026=`"Visual Studio 18 2026"` |
| `C4819` 警告 + 连锁错误 | 源码含中文注释，必须加 `/utf-8` 编译标志 |

## 9. 参考

### 管道命令参考

引擎通过 `\\.\pipe\AES67Engine` 接收文本命令：

| 命令 | 参数 | 说明 |
|------|------|------|
| `STATUS` | — | 查询引擎状态和指标 |
| `START` | — | 启动音频流 |
| `STOP` | — | 停止音频流 |
| `SET dest <addr>` | IP 地址 | 修改 TX 目标地址 |
| `SET port <n>` | 端口号 | 修改 TX 目标端口 |
| `SET source <addr>` | IP 地址 | 修改 RX 源地址 |
| `SET sourceport <n>` | 端口号 | 修改 RX 源端口 |
| `GET_ROUTING` | — | 获取当前路由表（JSON） |
| `SET_ROUTING <json>` | JSON | 更新路由表 |
| `EXIT` | — | 退出引擎进程 |

### 路由表 JSON 格式（`routing.json`）

```json
{
  "destinations": [
    {"name": "AES67 Stream 1", "address": "239.69.1.128", "port": 5004}
  ],
  "routes": [
    {"source": 0, "destination": 0, "gain": 1.85, "mute": false},
    {"source": 1, "destination": 0, "gain": 1.66, "mute": true}
  ]
}
```

- `source`：WASAPI 环回通道索引（0=左, 1=右）
- `destination`：目标流索引（对应 destinations 数组下标）
- `gain`：线性增益（1.0 = 0dB，0.5 = -6dB，2.0 = +6dB）
- `mute`：静音
