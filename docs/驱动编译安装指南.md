# AES67 虚拟声卡 — 驱动编译安装指南

> 内核骨架基于 MSVAD（微软音频驱动示例），源码已改造完成，位于本仓库 `driver/` 目录，无需再从上游 clone。
> 编译环境：Visual Studio + Windows WDK（版本按本机实际安装为准）。

---

## 1. 环境准备

| 组件 | 说明 |
|------|------|
| Visual Studio | 2022 (v17) 或 2026 (v18)，需装「使用 C++ 的桌面开发」工作负载 |
| Windows SDK | 随 VS 安装（版本号见 `C:\Program Files (x86)\Windows Kits\10\Include\` 下目录名） |
| Windows WDK | 与 SDK 同版本，https://aka.ms/wdk/windows11 |

**常见问题**：VS 装了但找不到内核头文件（`portcls.h`/`wdm.h`）→ WDK 没装或版本不匹配。检查 `C:\Program Files (x86)\Windows Kits\10\Include\<版本号>\km\` 是否存在。

> 源码文件已就位（`aes67driver.h` / `aes67driver.rc` / `AES67Driver.inf` / `AES67Driver.vcxproj` 等），**不需要**再做"从 Scream 改名/全局替换"这类历史步骤。若你是在 `feature/dev-fix` 分支上验证 P1，直接进第 2 节编译即可。

## 2. 编译

### 2.1 VS IDE 编译（推荐）

用 VS 直接打开 `driver\AES67Driver.vcxproj`，WDK 扩展会自动配好头文件路径。
在配置下拉框选 **`Win10` / `x64`**（不是 Debug/Release），Ctrl+Shift+B 编译。

### 2.2 命令行编译

仓库已带 `driver\build.bat`。**首次使用前**按脚本顶部注释改 2 处占位：

1. `WindowsSDKVersion` → 本机 WDK 版本号；
2. `vcvars64.bat` 路径 → 本机 VS 安装路径（VS2022 是 `\2022\`，VS2026 是 `\18\`）。

（脚本用 `%~dp0` 自动切到自身所在目录，把它放在 `driver\` 里双击即可，无需改目录。）

```bat
msbuild AES67Driver.vcxproj /p:Configuration=Win10 /p:Platform=x64 /t:Build
```

**注意**：`Configuration=Win10` 是项目里定义的配置名（非 `Debug`）。若报 `MSB8013`（配置组合不存在）就是配置名写错了。

产物路径：`driver\x64\Win10\`，包含 `AES67Driver.sys` / `AES67Driver.inf` / `AES67Driver.cat`（若 VS 自动生成）。

## 3. 签名

第三方内核驱动必须签名才能加载。开发/自测走**测试签名**：

### 3.1 开启测试模式

管理员 CMD：
```cmd
bcdedit /set testsigning on
```
重启。若报"安全引导策略保护"→ 进 BIOS 关闭 Secure Boot。

### 3.2 装测试证书

```cmd
cd /d <驱动目录>\x64\Win10
certutil -addstore Root AES67Driver.cer
certutil -addstore TrustedPublisher AES67Driver.cer
```

### 3.3 生成并签 Catalog

```cmd
"C:\Program Files (x86)\Windows Kits\10\bin\<SDK版本>\x86\inf2cat.exe" /driver:. /os:10_19H1_X64
"C:\Program Files (x86)\Windows Kits\10\bin\<SDK版本>\x64\signtool.exe" sign /fd SHA256 /a AES67Driver.cat
```

**常见错误**：
- `signtool` 不在 PATH → 用完整路径（`<SDK版本>` 换成本机 bin 下的实际目录名）
- `/a /f AES67Driver.cer` 失败 → `.cer` 只有公钥，用 `/a` 让工具自动从证书存储选

---

## 4. 安装驱动

```cmd
pnputil /add-driver <驱动目录>\x64\Win10\AES67Driver.inf /install
```

**常见错误**：
- "第三方 INF 不包含数字签名" → 没签 Catalog，回到 §3.3
- "拒绝访问" → 以管理员身份运行 CMD
- `devcon install` 失败且无提示 → 换 `pnputil`，报错更清楚

## 5. 验证

```cmd
pnputil /enum-devices /class MEDIA | findstr AES67
```

应看到 **1 个** `AES67Driver (WDM)` 设备。

Windows「声音设置 → 播放设备」列表里应出现 **AES67Driver**（一个播放/渲染端点）。

> 说明（P1 后现状）：当前 `.inf` 只声明了 `KSCATEGORY_RENDER`（播放端点）+ Topology，**尚无录音/捕获端点**——捕获端点属于内核收流（计划 P8）。
> 另外，P1 已把内核发送引擎掏空为**哑 sink**：驱动会正常呈现为可选的播放设备、接受音频写入，但**不再向网络发送任何数据**（样本被丢弃）。这是预期行为——网络发流已上移到用户态引擎，内核到用户态的数据通路（IOCTL + 共享内存）计划在 P9 打通。因此此阶段用 Wireshark 是抓不到组播流的，验证目标仅为"驱动能编译、加载、注册出播放端点"。

## 6. 卸载

```cmd
devcon remove *AES67Driver
pnputil /delete-driver oem<编号>.inf
```

**OEM 编号怎么找**：

1. 安装时的输出里就有——`发布名称: oem175.inf`，编号就是 `175`
2. 忘了的话用 `pnputil /enum-drivers` 列出所有已装驱动，搜 `AES67Driver` 对应的 `oem` 编号
3. 或直接去 `C:\Windows\INF\` 找最新的 `oem*.inf`，记事本打开确认是不是 AES67Driver

## 7. 踩过的坑

| # | 问题 | 根因 | 解决方法 |
|------|------|------|------|
| 1 | `portcls.h` 找不到 | WDK 没装/版本不匹配 | 装对应版本 WDK，检查 `Include\<版本>\km\` |
| 2 | `MSB8013: 配置和平台组合 Debug\|x64 不存在` | 项目用 `Win10` 而非 `Debug` 配置名 | `msbuild /p:Configuration=Win10` |
| 3 | `MSB8036: 找不到 Windows SDK 版本` | 项目文件里的 SDK 版本跟本机不对 | 用 VS 打开时会提示"重定目标"，选本机版本；或改 `.vcxproj` 里的 SDK 版本号 |
| 4 | `DIRID '10' not allowed` | 新 WDK 限制 INF 目录标识符 | INF 已用 `DestDir=12`（本仓库已修正） |
| 5 | testsigning 被 Secure Boot 锁 | BIOS Secure Boot 优先于 OS 策略 | 关 BIOS Secure Boot |
| 6 | "第三方 INF 不包含数字签名" | 只签了 .sys，INF 没签 | `inf2cat` + `signtool` 签 .cat |
| 7 | `signtool /a /f xxx.cer` 无效 | .cer 只有公钥无私钥 | 用 `/a` 自动从存储选证书 |
