# AES67 虚拟声卡 — 驱动编译安装指南

> 基于 Scream/MSVAD 骨架改造，编译环境 VS 2026 (v18) + WDK 10.0.28000.0

---

## 1. 环境准备

| 组件 | 版本 | 来源 |
|------|------|------|
| Visual Studio | 2026 (v18) | `D:\Program Files\Microsoft Visual Studio\18\Community` |
| Windows SDK | 10.0.28000.0 | 随 VS 安装，或单独下载 |
| Windows WDK | 10.0.28000.0 | https://aka.ms/wdk/windows11 |
| Scream 源码 | — | `git clone https://github.com/duncanthrax/scream.git` |

**常见问题**：VS 装了但找不到内核头文件（`portcls.h`/`wdm.h`）→ WDK 没装或装了低版本。检查 `C:\Program Files (x86)\Windows Kits\10\Include\10.0.28000.0\km\` 是否存在。

---

## 2. 驱动骨架改造

### 2.1 文件重命名

| Scream 原名 | AES67 改后 |
|------|------|
| `scream.h` | `aes67driver.h` |
| `scream.rc` | `aes67driver.rc` |
| `Scream.inf` | `AES67Driver.inf` |
| `Scream.vcxproj` | `AES67Driver.vcxproj` |

### 2.2 全局替换

将所有源文件（`.cpp`/`.h`/`.inf`/`.vcxproj`/`.rc`）中的 `Scream` → `AES67`、`SCREAM` → `AES67`。

### 2.3 项目文件 SDK 版本

编辑 `.vcxproj`，将所有 SDK 版本号（如 `10.0.22000.0`）替换为已安装的 WDK SDK 版本号（查 `C:\Program Files (x86)\Windows Kits\10\Include\` 下的目录名）。

### 2.4 INF 文件关键修正

```
[Version] 节:
  + PnpLockdown=1                    ← WDK 11 新增要求
  CatalogFile=AES67Driver.cat         ← 改名

[SourceDisksFiles] 节:
  AES67Driver.sys=222                 ← 改名

[DestinationDirs] 节:
  AES67Driver.CopyList=12             ← DIIRD '10' 不可用，换 12
```

## 3. 编译

### 3.1 VS IDE 编译（推荐）

用 VS 直接打开 `.vcxproj`，WDK 扩展自动配好头文件路径。Ctrl+Shift+B 编译。

### 3.2 命令行编译

创建 `build.bat`:

```bat
set WDKCONTENTROOT=C:\Program Files (x86)\Windows Kits\10\
set WindowsSDKVersion=10.0.28000.0
call "D:\Program Files\Microsoft Visual Studio\18\Community\VC\Auxiliary\Build\vcvars64.bat"
msbuild AES67Driver.vcxproj /p:Configuration=Win10 /p:Platform=x64 /t:Build
pause
```

**注意**：`Configuration=Win10` 而非 `Debug`——Scream 项目用的是自定义配置名。如果编译报 `MSB8013` 就是配置名不对。

---

## 4. 签名

编译产出 `AES67Driver.sys` 由 VS 自动用测试证书签名，但 `.inf` 需要额外签名。

### 4.1 开启测试模式

管理员 CMD：
```cmd
bcdedit /set testsigning on
```
重启。若报"安全引导策略保护"→ 进 BIOS 关闭 Secure Boot。

### 4.2 装证书

```cmd
cd /d <驱动目录>\x64\Win10
certutil -addstore Root AES67Driver.cer
certutil -addstore TrustedPublisher AES67Driver.cer
```

### 4.3 生成并签 Catalog

```cmd
"C:\Program Files (x86)\Windows Kits\10\bin\10.0.28000.0\x86\inf2cat.exe" /driver:. /os:10_19H1_X64
"C:\Program Files (x86)\Windows Kits\10\bin\10.0.28000.0\x64\signtool.exe" sign /fd SHA256 /a AES67Driver.cat
```

**常见错误**：
- `signtool` 不在 PATH → 用完整路径
- `/a /f AES67Driver.cer` 失败 → `.cer` 只有公钥，用 `/a` 自动从证书存储选

---

## 5. 安装驱动

```cmd
pnputil /add-driver <驱动目录>\x64\Win10\AES67Driver.inf /install
```

**常见错误**：
- "第三方 INF 不包含数字签名" → 没签 Catalog，回到 §4.3
- "拒绝访问" → 以管理员身份运行 CMD
- `devcon install` 失败无提示 → 换 `pnputil`，报错更清楚

---

## 6. 验证

```cmd
pnputil /enum-devices /class MEDIA | findstr AES67
```

应看到 3 个 `AES67Driver (WDM)` 设备。

Windows 声音设置 → 播放/录音设备列表里也应有 AES67Driver。

---

## 7. 卸载

```cmd
devcon remove *AES67Driver
pnputil /delete-driver oem<编号>.inf
```

**OEM 编号怎么找**：

1. 安装时的输出里就有——`发布名称: oem175.inf`，编号就是 `175`
2. 如果忘了，用 `pnputil /enum-drivers` 列出所有已装驱动，搜 `AES67Driver` 对应的 `oem` 编号
3. 或者直接去 `C:\Windows\INF\` 目录找最新的 `oem*.inf`，用记事本打开看是不是 AES67Driver

## 8. 踩过的坑

| # | 问题 | 根因 | 解决方法 |
|------|------|------|------|
| 1 | `portcls.h` 找不到 | WDK 没装/装错版本 | 装 WDK 10.0.28000.0，检查 `Include/km/` |
| 2 | `MSB8013: 配置和平台组合 Debug\|x64 不存在` | Scream 用 `Win10` 而非 `Debug` 配置名 | `msbuild /p:Configuration=Win10` |
| 3 | `MSB8036: 找不到 Windows SDK 版本` | 项目文件里的 SDK 版本跟本机不对 | 改 `.vcxproj` 里所有 SDK 版本号 |
| 4 | `DIRID '10' not allowed` | 新 WDK 限制 INF 目录标识符 | 换 `DestDir=12` |
| 5 | `scream.rc` not found | 文件改名后项目文件未更新 | `.vcxproj` 里改引用 |
| 6 | testsigning 被 Secure Boot 锁 | BIOS Secure Boot 优先于 OS 策略 | 关 BIOS Secure Boot |
| 7 | "第三方 INF 不包含数字签名" | 只签了 .sys，INF 没签 | `inf2cat` + `signtool` 签 .cat |
| 8 | `signtool /a /f xxx.cer` 无效 | .cer 只有公钥无私钥 | 用 `/a` 自动从存储选证书 |
