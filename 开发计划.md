# AES67 虚拟声卡 — 开发计划

> 版本 V1.0 | 目标：WDM + ASIO 双模虚拟声卡，驱动层基于 Scream/MSVAD 骨架改造

---

## 0. 架构总览

```
┌─────────────────────────────────────────────────────────┐
│                     Qt 配置面板 (用户态)                  │
│              设备发现 · 通道映射 · 状态监控               │
└──────────────────────────┬──────────────────────────────┘
                           │ Pipe/SharedMemory
┌──────────────────────────▼──────────────────────────────┐
│                AES67 音频引擎服务 (用户态)                │
│   RTP收发 │ PTPv2时钟 │ jitter buffer │ ASRC │ 混音     │
└────┬────────────────────────────────┬──────────────────┘
     │ IOCTL                          │ IOCTL
┌────▼──────────────┐     ┌───────────▼──────────────────┐
│   ASIO DLL        │     │    WDM 内核驱动 (Kernel)      │
│   (用户态)        │     │  Scream/MSVAD骨架改造          │
│   IOCTL直通引擎   │     │  Speaker + Mic Array + IOCTL  │
└───────────────────┘     └──────────────────────────────┘
```

**核心工程原则**：
- 内核驱动只做 buffer 搬运——零业务逻辑。崩了也不丢 AES67 状态
- 音频引擎全在用户态——可单独测试、可崩溃重启、可跨平台复用
- Qt 面板通过 Pipe/共享内存与引擎通信——面板崩溃不影响音频流
- ASIO 与 WDM 走同一个 IOCTL 通道——不重复实现

## 1. 模块拆分与依赖

```
M1. 驱动骨架      ← 依赖：无（只需 WDK + Scream 源码）
M2. IOCTL 通道    ← 依赖：M1
M3. 共享内存机制   ← 依赖：M1
M4. 用户态引擎框架 ← 依赖：无
M5. AES67 发送通路 ← 依赖：M3, M4
M6. AES67 接收通路 ← 依赖：M3, M4
M7. PTPv2 时钟    ← 依赖：M5, M6
M8. ASIO DLL      ← 依赖：M2, M4
M9. Qt 配置面板    ← 依赖：M4
```

**并行策略**：M1+M4 可同时开工（无依赖）。M5 和 M6 可并行。M7 在 M5/M6 完成后才做（需要实际流测试时钟精度）。

## 2. 开发里程碑

### M1 — 驱动骨架改造 (Scream → AES67 驱动)

**目标**：编译通过、安装后设备管理器出现虚拟声卡

| 任务 | 文件 | 说明 |
|------|------|------|
| 改名/清理 | `adapter.cpp`, `scream.h` | Scream → AES67Driver，清网络I/O残留 |
| 音频格式 | `wavtable.h` | Speaker 2CH L16/L24 48/96kHz + Mic 2CH |
| 拓扑 | `toptable.h` | Volume + Mute |
| INF | `AES67Driver.inf` | 修改设备名称、硬件ID、安装脚本 |

**验证**：`devcon install` 成功，设备管理器出现 "AES67 Virtual Audio"

### M2 — IOCTL 旁路通道

**目标**：用户态程序能通过 IOCTL 直通驱动的音频 buffer

| 任务 | 说明 |
|------|------|
| 定义 IOCTL 码 | `CTL_CODE(FILE_DEVICE_AUDIO, 0x800-0x80F, ...)` |
| 驱动侧 handler | 在 `adapter.cpp` 注册 IOCTL 回调 |
| 用户态测试 | 写一个极简 C 程序：`ioctl(hDevice, IOCTL_AES67_GET_BUFFER, ...)` |

**验证**：用户态能读到驱动 buffer 地址 + 当前读写位置

### M3 — 共享内存机制

**目标**：驱动和用户态引擎通过共享内存交换音频数据

| 任务 | 说明 |
|------|------|
| 驱动侧分配 | `MmAllocateNonCachedMemory` 分配无缓存内存 |
| 驱动侧映射 | 通过 IOCTL 返回共享内存句柄给用户态 |
| 用户态映射 | `MapViewOfFile` 映射到进程地址空间 |
| 环形 buffer | 实现 write/read 指针（驱动写、引擎读，或反过来） |

**性能约束**：
- 10ms buffer 周期，48kHz 2CH L24 = 2880 bytes/buffer
- 共享内存至少 4 倍 buffer 大小（~12KB）防止溢出
- 读写指针用 `volatile` + Memory Barrier——跨内核/用户态无锁

**验证**：用户态引擎能连续 30s 读写共享内存，无丢失

### M4 — 用户态引擎框架

**目标**：可启动、可停止、可测试的引擎骨架

| 组件 | 说明 |
|------|------|
| 主循环 | `while(running) { processAudio(); }` 线程 |
| 音频环回测试 | 从 WDM 播放拿数据 → 直接写回 WDM 录音（本地环回） |
| 日志/配置 | 命令行参数 + 文件日志 |
| 引擎生命周期 | Start/Stop/Pause/Resume 状态机 |

**架构要求**：
- 单进程、多线程：主线程（事件循环）+ 音频线程（实时，高优先级）
- 音频线程用 `AvSetMmThreadCharacteristics` 注册 MMCSS
- 所有内存预分配——音频线程不调用 `new/malloc`

**验证**：本地环回延迟 <10ms，30 分钟不掉帧

### M5 — AES67 发送通路

**目标**：WDM 播放 → 共享内存 → AES67 RTP 流 → 网络

| 任务 | 说明 |
|------|------|
| RTP 包封装 | RFC 3550 头 + AES67 载荷（L24 PCM, 48kHz, 1ms 包） |
| SAP 会话通告 | 周期性组播 SDP 描述 |
| UDP 发送 | `sendto()` 组播/单播 |
| 包速率控制 | 48kHz/48samples = 1000 pkt/s, 精确控制发送间隔 |

**工程约束**：
- RTP 包创建和发送用独立的非音频线程（避免阻塞音频采集）
- 音频线程只把 buffer 拷贝到发送队列

**验证**：Wireshark 抓包可识别为 AES67 流，Dante Controller 可发现

### M6 — AES67 接收通路

**目标**：网络 RTP → jitter buffer → 共享内存 → WDM 录音

| 任务 | 说明 |
|------|------|
| UDP 接收 | `recvfrom()` 收 RTP 包 |
| jitter buffer | 环形缓冲，包序号检测丢包/重排，动态窗口 2-6 包 |
| ASRC 校准 | `libsamplerate` 封装——时钟漂移时微调采样率 |
| 共享内存写入 | 音频线程从 jitter buffer 取数据，写入 WDM 录音 buffer |

**工程约束**：
- jitter buffer 大小自适应——测量网络抖动，动态调整窗口
- 丢包用静音填充——不爆音，不中断
- ASRC 调幅 ≤ 0.1%——人耳听不出

**验证**：从另一台 AES67 发送设备接收流，本地录音无爆音

### M7 — PTPv2 时钟

**目标**：软件从时钟，与 AES67 主时钟同步

| 任务 | 说明 |
|------|------|
| PTP 消息处理 | Sync/Follow_Up/Delay_Req/Delay_Resp |
| 软件时间戳 | `QueryPerformanceCounter` + `GetSystemTimePreciseAsFileTime` |
| 卡尔曼滤波 | 估算 offset + drift，滤除网络噪声 |
| 时钟调整 | 不调系统时钟——调整 ASRC 采样率（影响音频数据侧） |

**工程约束**：
- PTP 线程独立于音频线程
- 时钟同步精度 < 100μs（足够音频）
- 时钟源可配置为"本地自由时钟"（无 PTP 时降级运行）

**验证**：Wireshark PTP 插件显示 offset <1μs（统计平均）

### M8 — ASIO DLL

**目标**：DAW 可通过 ASIO 直接访问引擎

| 任务 | 说明 |
|------|------|
| ASIO SDK 集成 | `ASIOInit/ASIOStart/ASIOBufferSwitch` 实现 |
| IOCTL 直通 | ASIO DLL → IOCTL → 引擎共享内存（不经过音频栈） |
| 延迟报告 | `ASIOGetLatencies` 返回真实延迟 |

**验证**：Cubase/Reaper 识别设备，录音播放正常

### M9 — Qt 配置面板

**目标**：可视化设置界面

| 功能 | 说明 |
|------|------|
| 设备发现 | 显示网络中 AES67 设备 |
| 通道映射 | AES67 源 ↔ 本地通道 |
| 采样率/延迟 | 实时切换 |
| 状态监控 | 时钟同步状态、丢包数、buffer 水位 |
| 托盘图标 | 最小化到托盘 |

## 5. 代码质量准则

- 内核驱动：C 语言，不用 C++ 异常、不用标准库、不用动态内存（alloc once at init）
- 用户态引擎：C++17，RAII，智能指针，`std::vector`/`std::array` 代替裸指针
- 音频线程：无锁、无系统调用、无不预分配的内存操作
- 所有跨线程共享变量用 `std::atomic`
- 驱动和引擎间的共享内存用 Memory Barrier (`KeMemoryBarrier` / `std::atomic_thread_fence`)
- 日志分级（DBG/INFO/WARN/ERR），音频线程内不写详细日志
- 每个模块独立可测——引擎可以脱离驱动测、驱动可以脱离网络测
