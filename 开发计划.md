### 🧹 P1 — 清理驱动 Scream 残留　`✅ 完成`

现有 `driver/` 从 Scream/MSVAD 派生，带来大量与目标架构无关、甚至有害的历史残留。**及早清掉**，避免误导后来人以为"有一条共享内存通路 / 内核网络栈在工作"。

**现有代码起点**：`driver/`（`adapter.cpp`、`savedata.*`、`ivshmemsavedata.*`、`AES67Driver.inf` 等）。

| # | 现状残留 | 处理 |
|---|---|---|
| R-1 | 内核里跑 Scream 原有的 **WSK UDP 组播**（硬编码 `239.255.77.77:4010`），与引擎/ASIO 的 `239.69.1.128` 打架 | 删除内核网络 I/O 逻辑（`savedata.cpp` 的 WSK 组播） |
| R-2 | **ivshmem**（QEMU 虚拟机共享内存变体）与本机引擎无关 | 删除 `ivshmemsavedata.*` 及其引用 |
| R-3 | 换皮标识：池标签 `'DVSM'`、作者 `Marco Martinelli`/`Tom Kistner`、INF Provider、DriverVer 停在 2007 | 统一改为 AES67 项目标识 / 版本 |
| R-4 | `g_SharedBuffer` 只分配 + 清零、从不写音频（死缓冲）；IOCTL 码与 `ioctl_test.cpp` 对不上（`0x22E0000` vs `0x222003`）；`SET_FORMAT`/`GET_POSITION` 空壳 | 死缓冲与空壳 IOCTL 暂保留为"待 P9 实现"的占位，但**在代码中明确标注 TODO / 未实现**，不再让它伪装成可用通路 |

**说明**：R-4 里"共享内存通路"本身是 P9 的终局设计，此阶段不实现它，只是**去掉伪装**——把死缓冲、错码、空壳明确标为未实现占位，剩余 Scream 网络/ivshmem 残留直接删除。清理后驱动的唯一职责回归为"制造一个虚拟声卡端点"。

**完成判据**：驱动能编译、加载，注册出虚拟声卡端点；内核内不再有网络 I/O 与 ivshmem；`grep` 无 Scream 原作者标识残留。

**实际完成情况（已合入 main，tag `p1-done`）**：
- **R-1 已做**：`savedata.cpp/.h` 里整套 WSK UDP 组播发包引擎删除（socket/IRP/MDL/ring-buffer/WskRegister…）。`CSaveData` 掏空为**哑 sink**——公共接口签名（`Initialize/Disable/WriteData/WaitAllWorkItems/DestroyWorkItems/SetDeviceObject/GetDeviceObject`）全部保留，`WriteData()` 直接丢弃样本，`minstream.cpp`/`common.cpp` 无需大改。`adapter.cpp` 里网络相关全局变量与注册表读取（Unicast/DSCP/TTL/UseIVSHMEM）全删，只留 `SilenceThreshold`/`Version`。
- **R-2 已做**：删除 `ivshmemsavedata.cpp/.h` 整个文件；清除 `minstream.*`/`common.cpp`/`minwave.cpp`/`.vcxproj`/`.filters` 里所有 `g_UseIVSHMEM`/`CIVSHMEMSaveData` 引用（silence 检测逻辑保留，与网络无关，只是去掉一层分支缩进）。
- **R-3 已做**：PoolTag `'DVSM'`(显示 MSVD)→`'76SA'`(显示 AS67)；`.rc` 的 `scream.sys`/Tom Kistner 版权→ `aes67.sys`/项目署名；`.inf` Provider/Mfg `Tom Kistner`→项目名、DriverVer `2007`→当前日期+`0.1.0.0`、段名 `MicrosoftDS`→`AES67DS`；`.filters` 的死引用 `scream.h`/`scream.rc`/`netiodef.h` 清理。上游 MSVAD/微软源文件头版权声明按开源惯例**保留**。
- **R-4 已做（标注占位，不删）**：`g_SharedBuffer` 死缓冲、`AES67DeviceControl` 空壳 IOCTL、`g_AES67SymbolicLink` 未用符号链接、以及 IOCTL 码 `METHOD_NEITHER` 与 handler 里 `SystemBuffer` 用法不一致的隐患——均加 `TODO(P9)` 明确标注，交给 P9 打通主动脉时处理。
- **量级**：净删 ~1250 行内核网络代码（13 改 + 2 删）。
- **Windows 实测已通过**：`build.py driver` 编译干净；驱动加载不蓝屏；Windows 声音设置里出现 `AES67Driver` 播放端点。三项完成判据全部达成。
- **加载阶段修的两个坑**：
  - **0xFC 蓝屏**：`AES67DeviceControl`（运行时 IOCTL 回调）被 `#pragma code_seg("INIT")` 放进 INIT 段，而 INIT 段 discardable、`DriverEntry` 返回后被系统释放，首个 IOCTL 跳到已释放内存 → `ATTEMPTED_EXECUTE_OF_NOEXECUTE_MEMORY`。修法：把它及依赖移到 `PAGE` 段，`DriverEntry` 前切回 `INIT`。经验：**运行时回调（MajorFunction/DPC/工作项/定时器）绝不能放 INIT 段**。
  - **C3861 `MmGetPhysicalAddress` 未声明**：加 `#include <ntddk.h>` 无效（排在 portcls.h/wdm.h 之后，条件宏挡住声明，且两者混用有重定义风险）。该调用只在 P9 空壳 handler 里、`g_SharedBuffer` 恒为 NULL 走不到，故移除 ntddk.h、物理地址暂填 0 标 `TODO(P9)`。

## 4. 进度维护约定（贡献者必读）

为让任何接手者都能一眼看清"开发到哪了"，进度以**仓库自身**为准，不依赖聊天 / 口头交接。三层机制：

**① 进度总表（§0）—— 全局视图**
每次开始或完成一个阶段，更新 §0 表格的状态（✅/🚧/⬜）与对应阶段标题的徽标。

**② git tag —— 版本快照**
每个阶段验证通过、合入 `main` 后，在该 commit 打 tag：
```
git tag -a p1-done -m "P1 done: driver Scream residue cleaned"
git push origin p1-done
```
接手人 `git tag` 一看，**最后一个 `pN-done` 之后的提交就是当前在做的阶段**。

**③ commit / 代码标记 —— 细粒度**
- commit message 以阶段前缀开头：`P1: remove WSK multicast from savedata.cpp`。
- 半成品 / 占位代码在源码里显式标注：`// TODO(P9): shared-memory path not implemented yet`、`// WIP(P5): mixing bus`。接手人 `grep "TODO(P"` 即可列出所有未竟项。

**④ `PROGRESS.md` —— 时间线日志（可选但推荐）**
每阶段完成追加一行：日期 / 阶段 / 对应 tag / 一句话结果。作为人类可读的变更时间线，比翻 git log 更快。

> 一句话：**表看全局、tag 找快照、grep 找残项。** 三者随代码走，交接零成本。
